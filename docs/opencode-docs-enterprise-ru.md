# Enterprise (Предприятие)

Безопасное использование OpenCode в вашей организации.

OpenCode Enterprise предназначен для организаций, которые хотят гарантировать, что их код и данные никогда не покинут их инфраструктуру. Это достигается за счет использования централизованной конфигурации, которая интегрируется с вашим SSO и внутренним AI шлюзом.

> **Примечание**: OpenCode не хранит ваш код или данные контекста.

Чтобы начать работу с OpenCode Enterprise:
1. Проведите внутреннюю пробу (trial) со своей командой.
2. [Свяжитесь с нами](mailto:contact@anoma.ly), чтобы обсудить цены и варианты внедрения.

## Пробная версия (Trial)

OpenCode является открытым исходным кодом и не хранит ваш код или данные контекста, поэтому ваши разработчики могут просто [начать работу](opencode-docs-ru.md) и провести тестирование.

### Обработка данных

OpenCode не хранит ваш код или данные контекста. Вся обработка происходит локально или через прямые вызовы API к вашему AI провайдеру.
Это означает, что пока вы используете провайдера, которому доверяете, или внутренний AI шлюз, вы можете безопасно использовать OpenCode.

Единственным предостережением здесь является опциональная функция `/share`.

#### Шаринг разговоров

Если пользователь включает функцию `/share`, разговор и связанные с ним данные отправляются в сервис, который мы используем для размещения этих страниц шаринга на opencode.ai.

Данные в настоящее время обслуживаются через edge network нашей CDN и кэшируются на edge (краю сети) рядом с вашими пользователями. We recommend you disable this for your trial.

Мы рекомендуем отключить это для вашего тестирования.

```json
{
  "$schema": "https://opencode.ai/config.json",
  "share": "disabled"
}
```

[Узнайте больше о шаринге](opencode-docs-share-ru.md).

### Владение кодом

Вы владеете всем кодом, созданным OpenCode. Нет никаких лицензионных ограничений или претензий на владение.

## Развертывание (Deployment)

После завершения пробного периода и готовности использовать OpenCode в вашей организации, вы можете [связаться с нами](mailto:contact@anoma.ly), чтобы обсудить цены и варианты внедрения.

### Центральная конфигурация (Central Config)

Мы можем настроить OpenCode на использование единой центральной конфигурации для всей вашей организации.
Эта централизованная конфигурация может интегрироваться с вашим провайдером SSO и гарантирует, что все пользователи имеют доступ только к вашему внутреннему AI шлюзу.

### Интеграция SSO

Через центральную конфигурацию OpenCode может интегрироваться с провайдером SSO вашей организации для аутентификации.
Это позволяет OpenCode получать учетные данные для вашего внутреннего AI шлюза через вашу существующую систему управления идентификацией.

### Внутренний AI шлюз

С помощью центральной конфигурации OpenCode также можно настроить на использование только вашего внутреннего AI шлюза.
Вы также можете отключить всех других AI провайдеров, гарантируя, что все запросы проходят через одобренную инфраструктуру вашей организации.

### Самостоятельный хостинг (Self-hosting)

Хотя мы рекомендуем отключать страницы шаринга, чтобы гарантировать, что ваши данные никогда не покинут вашу организацию, мы также можем помочь вам разместить их на вашей инфраструктуре (self-host).
В настоящее время это находится в нашей roadmap. Если вы заинтересованы, [дайте нам знать](mailto:contact@anoma.ly).

**Безопасно ли это?**
Да. OpenCode не хранит ваш код или данные контекста. Вся обработка происходит локально или через прямые вызовы API к вашему AI провайдеру. С центральной конфигурацией и интеграцией SSO ваши данные остаются в безопасности в инфраструктуре вашей организации.

**Работает ли это с приватными npm реестрами?**
OpenCode поддерживает приватные npm реестры через встроенную поддержку файла `.npmrc` в Bun. Если ваша организация использует приватный реестр, такой как JFrog Artifactory, Nexus или аналогичный, убедитесь, что разработчики аутентифицированы перед запуском OpenCode.

Чтобы настроить аутентификацию с вашим приватным реестром:

```bash
npm login --registry=https://your-company.jfrog.io/api/npm/npm-virtual/
```

Это создает `~/.npmrc` с деталями аутентификации. OpenCode автоматически подхватит это.

> **Осторожно**: Вы должны войти в приватный реестр перед запуском OpenCode.

Альтернативно, вы можете вручную настроить файл `.npmrc`:

```bash
registry=https://your-company.jfrog.io/api/npm/npm-virtual/
//your-company.jfrog.io/api/npm/npm-virtual/:_authToken=${NPM_AUTH_TOKEN}
```

Разработчики должны войти в приватный реестр перед запуском OpenCode, чтобы гарантировать, что пакеты могут быть установлены из вашего корпоративного реестра.
