# Сеть (Network)

Настройка прокси и пользовательских сертификатов.
OpenCode поддерживает стандартные переменные окружения прокси и пользовательские сертификаты для корпоративных сетевых окружений.

## Прокси

OpenCode учитывает стандартные переменные окружения прокси.

```bash
# HTTPS прокси (рекомендуется)
export HTTPS_PROXY=https://proxy.example.com:8080 

# HTTP прокси (если HTTPS недоступен)
export HTTP_PROXY=http://proxy.example.com:8080 

# Обход прокси для локального сервера (обязательно)
export NO_PROXY=localhost,127.0.0.1
```

> **Осторожно**: TUI общается с локальным HTTP-сервером. Вы должны обходить прокси для этого соединения, чтобы предотвратить циклы маршрутизации.

Вы можете настроить порт и хостнейм сервера, используя [флаги CLI](opencode-docs-cli-ru.md#run).

### Аутентификация

Если ваш прокси требует базовой аутентификации, включите учетные данные в URL.

```bash
export HTTPS_PROXY=http://username:password@proxy.example.com:8080
```

> **Осторожно**: Избегайте жесткого кодирования паролей. Используйте переменные окружения или безопасное хранилище учетных данных.

Для прокси, требующих расширенной аутентификации, такой как NTLM или Kerberos, рассмотрите возможность использования LLM шлюза, который поддерживает ваш метод аутентификации.

## Пользовательские сертификаты

Если ваше предприятие использует пользовательские CA (Certification Authorities) для HTTPS соединений, настройте OpenCode, чтобы доверять им.

```bash
export NODE_EXTRA_CA_CERTS=/path/to/ca-cert.pem
```

Это работает как для прокси-соединений, так и для прямого доступа к API.
