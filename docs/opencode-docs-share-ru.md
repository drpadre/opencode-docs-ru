# Шаринг (Share)

Делитесь своими разговорами OpenCode.

## Как это работает (How it works)

Когда вы делитесь разговором, OpenCode:
1.  Создает уникальный публичный URL для вашей сессии.
2.  Синхронизирует историю вашего разговора с нашими серверами.
3.  Делает разговор доступным по ссылке для шаринга — `opncd.ai/s/<share-id>`.

## Режимы (Sharing modes)

OpenCode поддерживает три режима шаринга:

### Вручную (Manual - по умолчанию)
Сессии не шарятся автоматически, но вы можете поделиться ими вручную, используя команду `/share`.

```bash
/share
```

Чтобы явно установить ручной режим в вашем конфиге:
```json
{ "share": "manual" }
```

### Автоматически (Auto-share)
Вы можете включить автоматический шаринг для всех новых разговоров.

```json
{ "share": "auto" }
```

### Отключено (Disabled)
Вы можете полностью отключить шаринг.

```json
{ "share": "disabled" }
```

Чтобы принудительно применить это для всей команды, добавьте это в `opencode.json` в вашем проекте и закоммитьте в Git.

## Отмена шаринга (Un-sharing)

Чтобы прекратить делиться разговором и удалить его из публичного доступа:

```bash
/unshare
```

## Приватность (Privacy)

### Хранение данных (Data retention)
Общие разговоры остаются доступными до тех пор, пока вы явно не отмените их шаринг. Это включает:
*   Полную историю разговоров
*   Все сообщения и ответы
*   Метаданные сессии

### Рекомендации
*   Делитесь только теми разговорами, которые не содержат конфиденциальной информации.
*   Избегайте шаринга разговоров с проприетарным кодом или конфиденциальными данными.
*   Для чувствительных проектов отключайте шаринг полностью.

## Для предприятий (For enterprises)

Для корпоративных развертываний функция шаринга может быть:
*   Отключена полностью для соответствия требованиям безопасности.
*   Ограничена только пользователями, аутентифицированными через SSO.
*   Размещена на вашей собственной инфраструктуре (self-hosted).

[Узнайте больше](opencode-docs-enterprise-ru.md) об использовании OpenCode в вашей организации.
